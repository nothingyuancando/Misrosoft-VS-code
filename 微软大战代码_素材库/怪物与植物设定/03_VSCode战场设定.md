# 03 VSCode 战场设定

战斗地点可以选在 VSCode 上，而且这个选择很适合游戏主题。

## 为什么适合

VSCode 本身就是程序员工作场景。把塔防战场放进编辑器后，玩家会天然理解：

```txt
左侧资源管理器：你的项目文件
中间编辑器：敌人入侵主战场
底部终端：战报和错误日志
顶部标签页：当前关卡文件
右侧迷你地图：敌人波次预警
```

这样游戏不是在普通草坪上防守，而是在保护自己的项目。

## 交互区域

### 左侧 Explorer

放卡牌、资源和当前选择。

```txt
coffee
print
if wall
bug
rollback
function cannon
```

### 中间 Editor

放 5 x 9 战场。格子可以设计成代码行。

每条路线对应一个进程：

```txt
browser.process
editor.process
terminal.process
database.process
slackoff.process
```

### 底部 Terminal

显示战报：

```txt
> print 炮输出了一颗字符弹
> Teams 会议怪发起会议
> try-catch 捕获异常
```

### 右侧 Minimap

显示波次预警、boss 血条、更新进度。

## 视觉风格

建议使用 VSCode 深色主题。

颜色方向：

```txt
背景：#1e1e1e
侧栏：#252526
活动栏：#333333
蓝色强调：#007acc
终端绿色：#89d185
警告黄色：#cca700
错误红色：#f14c4c
```

这些颜色能和蓝屏、代码、终端日志形成统一感。
