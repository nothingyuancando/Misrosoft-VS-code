# 04 贴图拼贴规范

原型阶段可以把梗图贴到怪物头上，但要有规则。随意拼贴会很好笑，也会让玩家看不清战场。

## 基本原则

```txt
身体统一，贴纸变化
剪影优先，梗图其次
技能前摇必须清楚
贴纸统一白边和阴影
截图只裁关键区域
```

## 怪物拼贴规则

### 头部贴纸

头部贴纸大小建议占怪物高度的 35% 到 45%。不能盖住身体动作。

适合放在头上的元素：

```txt
Windows 更新进度条
Edge 图标
Teams 会议通知
OneDrive 云
蓝屏 :( 表情
Clippy 眼睛
99% 进度数字
```

不建议整张截图直接贴上去。截图要裁成小贴纸。

### 身体剪影

身体决定玩法：

```txt
小兵：普通怪
小车：快怪
气泡：控制怪
云朵：分裂怪
巨人：boss
弹窗：干扰怪
眼睛：侦察或复制怪
```

玩家远看只看剪影，也应该知道大概威胁。

### 技能前摇

每个怪的技能释放前要有明显提示。

```txt
更新兵：进度条变满
Edge 传教士：举起 Set default? 牌子
Teams 怪：地上出现会议圈
OneDrive 怪：背后出现冲突副本
蓝屏巨人：胸口滚动 stop code
Clippy：弹出 Need help?
```

## 植物拼贴规则

基础植物要比怪物更稳定。它们是玩家的工具，不要太乱。

```txt
coffee 机：咖啡机主体
print 炮：控制台窗口炮塔
if 墙：条件门
bug 雷：错误图标地雷
rollback：倒带箭头炮
函数炮：坐标纸炮台
```

逻辑模块必须像插件：

```txt
for：旋转环
if：筛选框
else：备用弹簧板
try-catch：花括号护盾
cache：缓存盒
sudo：权限徽章
```

## 资源替换规则

同名文件可以替换。例如后续你找到更好的 Edge 梗图，可以直接替换：

```txt
图片资源/怪物贴纸/edge_head.svg
```

代码里只引用文件名，不关心图片内部是什么。

## 第一版建议

第一版不要追求素材全是真实截图。优先保证：

```txt
看得清
能区分
好笑
能快速实现
```

真实截图适合放进怪物头部、战报弹窗和梗库页面。战斗中的角色最好用转绘贴纸，风格会更统一。
