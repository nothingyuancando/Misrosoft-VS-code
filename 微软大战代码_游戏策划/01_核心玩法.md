# 01 核心玩法

## 游戏类型

《微软大战代码：请不要关闭你的电脑》是一个横版路线塔防。玩家在左侧放置代码植物，敌人从右侧进入。每一行是一条系统进程路线，敌人走到最左侧时会破坏玩家的项目文件。

基础玩法接近《植物大战僵尸》，但核心差异是代码组合。玩家先用咖啡种下基础代码，再用内存在同一格或相邻格叠加逻辑模块，让植物获得新的行为。

## 战场结构

推荐第一版地图：

```txt
5 行 x 9 列

浏览器进程    [ ][ ][ ][ ][ ][ ][ ][ ][ ] ← 敌人
编辑器进程    [ ][ ][ ][ ][ ][ ][ ][ ][ ] ← 敌人
终端进程      [ ][ ][ ][ ][ ][ ][ ][ ][ ] ← 敌人
数据库进程    [ ][ ][ ][ ][ ][ ][ ][ ][ ] ← 敌人
摸鱼进程      [ ][ ][ ][ ][ ][ ][ ][ ][ ] ← 敌人
```

左侧 6 列是主要种植区。右侧 3 列是敌人生成与推进区。敌人从右向左移动，代码植物一般向右攻击。

## 胜负条件

胜利条件：

```txt
撑过全部波次
打败最终 boss
main 分支没有被攻破
```

失败条件：

```txt
任意一行项目文件被连续攻破 3 次
或者 boss 抵达最左侧
```

失败提示可以做成文件损坏梗：

```txt
项目已被移动到 OneDrive。
你没有权限访问此文件。
```

或者：

```txt
毕业设计-最终版-真的最终版.zip 已损坏。
```

## 局内循环

每局由 5 到 8 波敌人组成。第一版建议 5 波。

基础循环：

```txt
1. coffee 机生产咖啡
2. 玩家用咖啡种植基础代码植物
3. 玩家用内存给已有植物叠加逻辑模块
4. 微软军团按波次从右侧出现
5. 代码植物自动攻击、阻挡、控制或生产资源
6. 玩家根据敌人类型调整防线
7. 最后一波出现蓝屏巨人
```

## 操作方式

玩家操作要保持简单。

```txt
左键点击卡牌
左键点击格子种植
点击已有植物后，可以选择叠加模块
右键或长按查看植物组合说明
```

组合不要让玩家输入代码。所有组合都通过卡牌和模块完成。

## 游戏节奏

推荐一局 4 到 6 分钟。

前 60 秒教玩家种 coffee 机、print 炮和 if 墙。中段引入 Edge 传教士、Teams 会议怪、OneDrive 同步怪。最后用蓝屏巨人测试玩家是否理解 try-catch、rollback 和组合输出。

## 战报系统

右侧显示实时战报。战报不只是装饰，它要解释敌人技能和植物组合。

示例：

```txt
print 炮输出了一颗字符弹。
for 模块接管 print 炮，开始连续输出。
Teams 会议怪拉起会议，for 循环枪被迫同步进度。
try-catch 捕获了会议邀请。
Edge 传教士正在修改默认浏览器。
sudo 锤拒绝了权限请求。
```

战报的目标是让玩家知道发生了什么，同时保留恶搞感。

## 特色机制

本游戏的核心特色是「代码逻辑叠加」。基础植物像代码语句，逻辑模块像控制结构。

示例：

```txt
print
基础效果：发射一颗子弹

print + for
组合效果：连续发射子弹

print + if
组合效果：只攻击满足条件的敌人，例如优先打快怪

print + for + try-catch
组合效果：连续发射，且第一次异常不会停火
```

这样玩家不需要写代码，也能感受到写代码的逻辑。
