# 05 关卡与事件

## 关卡设计目标

当前原型先做单局生存，不做多关卡章节。

一局要逐步引入：

```txt
移动躲避
coffee 拾取
植物武器升级
锦囊手牌救场
弹窗干扰
文件生命压力
Boss 战
```

## 单局时间轴

建议一局 3 到 5 分钟。

### 0:00 - 0:30 开局适应

目标：

```txt
让玩家理解自己控制 VS Code 图标
看到 print 炮自动开火
拾取第一批 coffee
```

敌人：

```txt
Windows 更新
Edge 弹窗
Teams 会议
OneDrive 冲突
```

事件：

```txt
开局免费摸 2 张锦囊
默认拥有 print 炮 Lv.1
```

### 0:30 - 1:20 构筑成型

目标：

```txt
第一次或第二次升级
让玩家看到武器挂在主角周围
开始出现更明显的桌面事故
```

新增敌人：

```txt
YAML 缩进
Defender 误杀
```

推荐弹窗：

```txt
Windows Update
Edge
Teams
```

### 1:20 - 2:20 压力提升

目标：

```txt
让玩家必须使用快捷锦囊或手牌
让文件生命开始产生压力
```

新增敌人：

```txt
npm 黑洞
需求变更
```

事件节奏：

```txt
敌人刷出间隔缩短
弹窗出现频率提高
OneDrive 冲突副本更容易堆起来
```

### 2:20 - 3:00 Boss 前夜

目标：

```txt
让玩家尽量补构筑、补手牌、清弹窗
```

推荐事件：

```txt
CI Pipeline 类延时锦囊价值提高
Alt+F4 成为关键保命手段
文件 already deleted 风险明显上升
```

### 3:00 Boss：蓝屏合并冲突

Boss 行为：

```txt
高血量追逐玩家
周期性生成 BSOD 弹窗
周期性伤害随机文件
作为高威胁目标被 TypeScript / IDEA / kill -9 锁定
```

胜利条件：

```txt
击败 Boss
至少保留 1 个 safe 文件
```

胜利提示：

```txt
git commit 成功：VS Code 工作区挺过了微软桌面大乱斗。
```

## 全局事件

### Windows Update

```txt
表现：弹窗提示你的电脑将在 2 分钟后重启
处理：点击关闭或 Alt+F4
失败：伤害一个文件
```

### Edge 默认浏览器

```txt
表现：弹窗询问是否设为默认浏览器
处理：点击关闭、Alt+F4、击败 Edge 敌人
失败：伤害一个文件
```

### Teams 会议邀请

```txt
表现：你能听到我吗？
处理：点击关闭或用 Ctrl+S 冲击波清场
失败：伤害一个文件
```

### OneDrive 同步

```txt
表现：正在同步 决赛版-最终版.zip
处理：Docker、Alt+F4、快速清怪
失败：污染一个文件
```

### BSOD

```txt
表现：CRITICAL_PROCESS_DIED
处理：优先点击关闭或 Alt+F4
失败：高额文件伤害
```

## 节奏控制

第一版节奏原则：

```txt
前 30 秒不要太难，先让玩家移动和捡 coffee
1 分钟后开始要求玩家使用手牌
2 分钟后允许文件被打出 already deleted
3 分钟 Boss 必须明显改变战斗气氛
```

```txt
依赖黑洞

```
