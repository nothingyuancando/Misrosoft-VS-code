# 02 资源与组合系统

## 资源设计

游戏有两种主要资源：

```txt
咖啡：用于种植基础代码植物
内存：用于叠加逻辑模块和升级组合
```

咖啡对应《植物大战僵尸》里的阳光。它让玩家扩展防线。

内存是本游戏的特色资源。它不直接种出新植物，而是让已有代码植物变复杂。玩家可以把 `for`、`if`、`try-catch`、`async` 之类的逻辑模块叠到已有植物上，改变植物行为。

## 咖啡

咖啡来源：

```txt
coffee 机定期生产
击败部分敌人掉落
关卡事件奖励
Stack Overflow 祭坛偶尔提供
```

咖啡用途：

```txt
种植基础植物
补种被摧毁的防线
释放少量低级技能
```

第一版建议咖啡上限为 999，避免界面复杂。

## 内存

内存来源：

```txt
memory 条定期增长
cache 塔提供额外内存
击败精英怪掉落内存碎片
完成小目标获得内存包
```

内存用途：

```txt
给植物叠加逻辑模块
升级组合层级
修复异常状态
发动函数炮
```

内存比咖啡更稀缺。玩家应该经常做选择，是把 `print` 变成连发炮，还是给关键防线加 `try-catch` 保护。

## 种植和叠加

基础规则：

```txt
种植基础植物消耗咖啡
叠加逻辑模块消耗内存
每个格子最多有 1 个基础植物
每个基础植物最多叠加 2 个逻辑模块
高级关卡可以解锁第 3 个模块槽
```

示例：

```txt
print 炮
费用：100 咖啡
效果：每 1.5 秒发射 1 颗字符弹
```

```txt
for 模块
费用：40 内存
效果：让目标植物连续触发 3 次
```

组合后：

```txt
print + for
总费用：100 咖啡 + 40 内存
效果：每轮连续发射 3 颗字符弹
副作用：连续攻击 4 轮后过热，暂停 1 秒
```

## 组合槽位

每个基础植物有 2 个模块槽。

```txt
槽位 A：主逻辑
槽位 B：保护或修饰逻辑
```

推荐第一版不要让槽位规则太复杂。玩家只需要知道，先种植物，再把逻辑模块拖上去。

示例：

```txt
print + for + try-catch
```

含义：

```txt
print 负责发射
for 让它连发
try-catch 防止第一次异常打断输出
```

## 组合副作用

组合要强，但不能完全无脑。越复杂的代码越容易出现异常。

建议每个组合有三种风险：

```txt
过热：攻速太快导致短暂停顿
异常：被特殊怪干扰后停止工作
内存泄漏：持续占用内存，降低后续模块部署速度
```

示例：

```txt
print + for
强度：高频输出
风险：过热
```

```txt
regex + for
强度：连续控制
风险：匹配错误，可能缠住自己人
```

```txt
async + await
强度：范围时停
风险：被打断后出现 Promise 未处理异常
```

## 组合规则表

| 基础植物 | 叠加模块 | 组合名 | 效果 | 风险 |
| --- | --- | --- | --- | --- |
| print | for | for-print 循环炮 | 连续发射 3 颗子弹 | 过热停顿 |
| print | if | if-print 条件炮 | 优先攻击指定类型敌人 | 条件不满足时不攻击 |
| print | async | async-print 延迟炮 | 延迟后发射强力弹 | 可能打空 |
| print | try-catch | 稳定 print | 免疫一次打断 | 输出较慢 |
| if 墙 | else | if-else 双分支墙 | 第一层倒下后触发反击 | 反击后墙体变脆 |
| if 墙 | try-catch | 异常保险墙 | 抵消一次特殊技能 | 冷却较长 |
| bug 雷 | for | 连环 bug 雷 | 连续爆炸 3 次 | 可能提前引爆 |
| bug 雷 | rollback | 时空 bug 雷 | 爆炸并击退敌人 | 有概率击退自己植物 |
| regex 藤蔓 | for | 循环匹配藤蔓 | 多次定身 | 误伤概率提高 |
| rollback | if | 条件回滚器 | 只回滚精英怪或 boss 技能 | 对普通怪效率低 |
| coffee 机 | cache | 缓存咖啡机 | 首次产量翻倍 | 后续可能产出旧咖啡 |
| StackOverflow | copy | 复制答案祭坛 | 随机复制附近植物 buff | 可能复制过期答案 |

## 组合稀有度

组合可以分为三档。

```txt
基础组合：容易理解，数值稳定，例如 print + for
梗向组合：有趣但带副作用，例如 regex + for
危险组合：威力大但会误伤，例如 rm -rf + sudo
```

第一版优先做基础组合和少量梗向组合。危险组合放到后续版本。

## 内存压力

内存要让玩家产生取舍。

例如：

```txt
玩家有 80 内存
选择 A：给两个 print 炮各加一个 for 模块
选择 B：给一条关键防线加 try-catch 和 rollback
选择 C：攒到 120 内存放函数炮
```

这种选择会让同一张地图出现不同打法。

## 函数炮消耗

函数炮属于特殊组合。它可以用咖啡种下，但每次发射需要内存。

```txt
种植费用：200 咖啡
发射费用：60 内存
冷却时间：20 秒
```

玩家选择弹道：

```txt
y = 0        直线炮
y = x        斜线炮
y = -x       反向斜线炮
y = x*x      抛物线炮
y = sin(x)   波浪炮
y = random() 乱码炮
```

第一版可以先做按钮，不做自由输入。
