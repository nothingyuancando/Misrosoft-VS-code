# 微软大战代码：游戏策划总览

《微软大战代码：VS Code 桌面幸存者》是一个程序员梗向的桌面幸存者游戏。

玩家不再种植物守线，而是直接控制 VS Code 图标，在编辑器窗口里躲避微软弹窗和开发事故。代码植物变成围绕主角成长的自动武器；锦囊手牌保留三国杀式的瞬时救场；左侧 Explorer 文件是本局生命线，被打穿后只在游戏界面显示 `already deleted`。

## 一句话玩法

> 控制 VS Code 图标，在被 Windows 更新、Edge 弹窗、Teams 会议、OneDrive 冲突和蓝屏 Boss 污染的编辑器里生存，用植物武器和程序员锦囊守住项目文件，直到最终提交成功。

## 设计目标

- 跳出《植物大战僵尸》的格子守线框架，改成更爽快的移动生存。
- 保留“微软大战代码”的核心梗味：更新、弹窗、默认浏览器、同步冲突、误杀、蓝屏。
- 保留代码植物，但把它们变成武器构筑，而不是固定防御塔。
- 保留抽卡锦囊，让 `Ctrl+S`、`kill -9`、`Git Stash`、`Alt+F4` 成为关键时刻的爽点。
- 不做真实开发模拟，避免像在干活；程序员元素主要作为武器、敌人、弹窗和战斗文本。

## 当前可玩原型

当前 HTML 原型已经实现：

```txt
VS Code 主角
WASD / 方向键移动
植物武器自动攻击
coffee 掉落与升级三选一
主角身边显示武器图标和等级
上方锦囊手牌
Ctrl+S / kill -9 / Alt+F4 快捷技能
Explorer 文件生命
可点击关闭的微软弹窗
3 分钟蓝屏合并冲突 Boss
```

## 后续设计重点

下一阶段应该围绕“构筑爽感”和“桌面乱斗感”继续加强：

- 让更多植物武器拥有明显的屏幕表现。
- 增加武器进化，例如 `Docker + CI Pipeline`、`TypeScript + ESLint`。
- 让弹窗更像真实 Windows 桌面干扰，例如遮挡、拖拽、倒计时。
- 强化 Boss 阶段，让蓝屏、冲突标记、重启倒计时更有压迫感。
- 平衡 coffee 经济，避免抽卡和升级互相抢资源过重。
