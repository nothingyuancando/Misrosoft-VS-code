# 06 MVP 原型范围

## 原型目标

第一版原型只验证核心玩法，不追求内容量。

要验证的三件事：

```txt
1. 横版塔防是否顺手
2. 咖啡种植和内存组合是否有策略
3. 微软怪物和程序员战报是否足够有趣
```

## 必做内容

### 地图

```txt
5 行 x 9 列
左侧种植
右侧刷怪
每行代表一个进程
```

路线名：

```txt
浏览器进程
编辑器进程
终端进程
数据库进程
摸鱼进程
```

### 资源

```txt
咖啡：种植基础植物
内存：叠加逻辑模块、发射函数炮
```

第一版资源来源：

```txt
coffee 机产咖啡
每 10 秒自然恢复少量内存
击败精英怪掉落内存
```

### 基础植物

第一版做 6 个：

```txt
coffee 机
print 炮
if 墙
bug 雷
rollback 器
函数炮
```

### 逻辑模块

第一版做 4 个：

```txt
for
if
else
try-catch
```

### 组合

第一版至少实现 6 个组合：

```txt
print + for：连续发射
print + if：优先攻击快怪或精英怪
print + try-catch：免疫一次打断
if 墙 + else：墙破后反击
bug 雷 + rollback：爆炸并击退
rollback + if：优先回滚精英怪
```

如果时间允许，再加：

```txt
print + for + try-catch：稳定循环炮
bug 雷 + for：连环爆炸
```

### 敌人

第一版做 6 个：

```txt
Windows 更新兵
强制重启车
Edge 默认浏览器传教士
Teams 会议怪
OneDrive 同步怪
蓝屏巨人
```

### 波次

第一版 5 波：

```txt
第 1 波：Windows 更新兵
第 2 波：Windows 更新兵 + 强制重启车
第 3 波：Edge 传教士 + Windows 更新兵
第 4 波：Teams 会议怪 + OneDrive 同步怪
第 5 波：蓝屏巨人 + 强制重启车
```

### UI

需要的界面元素：

```txt
上方：咖啡、内存、当前波次、更新进度
中间：5 x 9 战场
下方：植物卡牌和模块卡牌
右侧：战报
```

战报第一版至少有这些事件：

```txt
植物种植
模块叠加
敌人出现
敌人释放技能
组合触发
植物被摧毁
敌人被击败
胜利或失败
```

## 可延后内容

这些不要放进第一版：

```txt
联网
存档
多关卡章节
自由输入函数
完整梗库
几十种植物
复杂剧情
装备系统
成就系统
```

先做一关。能玩、有笑点、能体现组合，就算成功。


## 原型验收标准

原型达到以下标准即可进入下一阶段：

```txt
玩家能完成一局 4 到 6 分钟的战斗
玩家能看懂咖啡和内存的区别
玩家能主动尝试 print + for 之类的组合
至少一个敌人技能能逼玩家改变摆法
战报中至少有 20 条有趣文本
蓝屏巨人作为 boss 有压迫感
```

## 下一阶段扩展

原型成立后，再加入：

```txt
更多微软梗怪物
更多代码模块
关卡章节
梗库收集
函数炮自由输入
植物升级树
隐藏事件
```
